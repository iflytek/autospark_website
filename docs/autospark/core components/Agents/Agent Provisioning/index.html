Agent Provisioning

介绍​

代理供应为您提供了创建新代理的功能。您可以广泛定制代理，并提供最符合您要求的代理。

名称和说明​

您可以首先为您的代理提供一个唯一的名称，并对代理的特征进行描述。这是为了便于识别，使您更容易了解与您合作的代理。

目标​

  • 目标是你可以为代理定义的理想结果或目的。您定义的目标是总体目标，而代理的行为正是基于这些目标。

  • 您可以用简单明了的英语逐步定义代理要实现的目标。您可以随意设计您的提示，并对目标进行试验，以便引导代理实现您所期望的结果。

说明​

  • 指令是你可以为代理人规定的指令或指南。通过指令，您可以提供可操作的信息，帮助代理人了解如何朝着既定目标前进，并做出适当的选择，帮助代理人朝着预期目标前进。

  • 值得注意的是，没有必要总是向代理人下达指示。但是，如果指示到位，它就会成为催化剂，使代理的绩效得到显著提高。

模型​

创建代理时，您可以选择要部署代理的大语言模型。

目前，autoSpark 支持 OpenAI 的 GPT 3.5-Turbo、GPT 3.5-Turbo-16k、GPT-4、GPT-4-32k 和 Google Palm Bison。

工具和工具包​

  • 您可以选择所需的工具包或工具，让代理与之互动并执行操作。使用工具包可以帮助您简化代理工作流程，取得更好的效果。

  • 您可以根据具体要求选择多个工具包或工具。要进一步了解 SuperAGI 工具包的工作原理，请访问工具包部分。

高级选项​

通过 autoSpark 中的高级选项，您可以根据自己的要求修改代理的行为，从而获得更多的实践机会。

代理类型​

选择一种代理类型，就可以改变代理的思维过程。例如基于 ReAct 的代理、基于固定任务的代理和基于动态任务的代理。要进一步了解每种代理类型的工作原理，可以浏览代理类型页面。

添加资源​

您可以添加 .txt、.pdf、.csv、.epub、.docx 等格式的资源，代理将在运行过程中摄取这些资源。要进一步了解资源管理器的工作原理，请浏览资源管理器部分。

制约因素​

  • 约束条件是代理必须在其中运行的特定护栏。这些制约因素可确保代理在设定的环境中运行，并遵守相关准则。

  • 在创建代理时，会预先定义约束条件。不过，您也可以根据需要更改约束条件，以更好地优化代理的行为。

最大迭代次数​

最大迭代次数是为代理设置的预定义迭代限制。由于运行 LLM 的成本可能会很高，而且代理可能会产生幻觉，因此您可以设置代理必须运行的迭代次数。

权限类型​

您可以在 "上帝模式 "和 "受限模式 "之间做出选择。"上帝模式 "是一种真正的自主代理，它可以根据您的目标和指令独立做出决定并执行任务；而 "受限模式 "则是一种在执行任务前需要征得您同意的代理。
